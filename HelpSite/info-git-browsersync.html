Git & GitHub

This guide assumes that Git, GitHub & BrowserSunc(through Node.js) is already set-up. I will only put some difficult to remember snippets & tid-bits here

Git & GitHub

  • Pull
    • Basically making a copy of your GitHub repository on your computer for editing, tracking & eventually re-uploading
    1. To pull a repository from GitHub, use cd(change Directory) and Dir(Directory List) to go about navigating to the file path you would like to pull the repo to within the comman prompt - This can be any file path - Windows Button + R, "CMD", Enter
    2. Get the URL/link to the repo, this is usually found on the top left of the repo on a button denoting 'Clone or Download this Repository'
    3. Run the the comman "git pull yoururlhere.com"
    4. The pull request will be processed and the repo will be downloaded to the directory of your chosing
  • Status
    • Most commonly used Git command. Checks the status of your repository on GitHub vs. the status of the cloned/pulled repository on your local machine. You will then see if there are 'Untracked' and/or 'Uncommited' files within your local repo to track, commit or push or the opposite if you were working on the GitHub repo from a different PC, or with someone else.
    1. simply ensure you are within the git repostory in command prompt
    2. Run the ommand 'git status'
  • Add
    • To add a file that you have worked on but have not told Git is part of your repository yet. It will not form part of the repository.
    1. Firstly ensure you run 'git status', in the repo file directory on command prompt
    2. run 'git add .'(To add ALL untracked files) or 'git add C:\Users\YourUsername\YourGitRepoDirectory\YourFileName.ext'. To clarify, I have used .ext here only as a placeholder, would be .html or .pdf etc.
    3. Your file(s) will now be tracked by git, thereby when running 'git status' you will be able to see if any changes were made etc.
  • Commit
    • When you use the commit command you are telling git that you would like to upload the tracked files(SO do this only after running 'git add') to your GitHub repository
    1. Ensure you are in the current repo directory in command prompt
    2. Run the commit command 'git commit -m "Your Reminder Message ABot What Changes or New files you are adding here"'. the '-m' part tells GitHub to include the message in the changelog.
    3. After this you can run 'git status' to ensure all your files are tracked, commited and ready for the push
  • Push
    • This is simply the command you will run to push/upload your changes, new files & media to GitHub
    1. Ensure you are in the current repo directory in command prompt
    2. Run the command 'git push'
    3. All the commited files will now be uploaded to GitHub and if you have GitHub pages running, you can open up your 'website' by getting your Pages link and navigating manually to all your various folders with index.html files.
    For more on Git & Github, please read the GitPro book

BrowserSync

BrowserSync is relatively simple to use albeit the command may be nigh impossible to rememeber so I will simply explain it in TWO(2) easy steps
  1. Ensure you are within the correct directory for your GitHub repos.(This is not a requirements as you can navigate within BrowserSync however it makes it a lot simpler).
  2. Run: browser-sync start --server --directory --files "*"
  3. It will open in chrome with a clickable directory to all the HTML files you have on your computer. If you navigated to he correct directory when running the command - it will simply show you the available html files within your dir. Double click & it will open the file and update everytime you save it within your IDE.
